Cyfer BLE Tx - BLE Format

Introduction

Using your watch's Bluetooth capabilities (BLE), Cyfer BLE Tx broadcasts your heart rate to devices such as gym and medical equipment. The heart rate readings are transmitted using an open standard so should work with most equipment without any configuration. Just tap the app's On button (middle of the screen), accept all the permissions and you will be broadcasting your heart rate. (While you are wearing your watch!)

Your watch battery level is also broadcast.

Optionally, your step count and GPS location can be broadcast, however, open standards for these readings do not exist so to detect these broadcasts will require specific Cyfer BLE Tx apps or your own code. See the BLE configuration section, below, for further details.

Note that the steps counter will only count steps while the app is running. This means the steps count will have a different value to your watch's built-in step counter which is usually configured to run continually.

If you wish to confirm that CyferBLE Tx is working correctly, you can install on your phone an app such as LightBlue or BLE Scanner (available for free on the Google Play Store).

Android Permissions

Several permissions need to be granted for the app to access and broadcast your heart beat readings. You will be prompted to grant them when you initiate the broadcasting. There are additional required permissions if you decide to broadcast your watch's GPS readings.

To run continuously in the background, the app needs the Alarms & Reminders permissions. If you will only be running the app for a few minutes or hours, then you don't need to give this permission. However, if you want the app to run continuously, reliably (i.e. not being shutdown by Android), then authorising the permission will be required. Some watches have a bug where the user is not able to give this permission. If this is the case for your watch, then you will either have to accept the app will randomly stop (especially during charging) and remember to check that the app doesn't need to be restarted, or use a route for enabling the permission which bypasses the bug, but is quite technically involved. Please Google or use AI for further information on how to do this but basically you need to use the Android developers tool adb (Android Debug Bridge) as follows:

adb shell appops set --uid uk.cyferltd.cyferblefull SCHEDULE_EXACT_ALARM allow

And then to test the command has worked, you can use:

adb shell appops get uk.cyferltd.cyferblefull SCHEDULE_EXACT_ALARM

The output should look similar to:

Uid mode: SCHEDULE_EXACT_ALARM: allow

SCHEDULE_EXACT_ALARM: allow

BLE configuration

The following is provided in case you wish to programmatically access the broadcast information. All data types, presented below, are Java data types

Heart rate

Service: 180d

Measurement: 2a37

Data format: 1xShort

Battery level

Service: 180f

Measurement: 2a19

Data format: 1xByte

Steps

Service: fee7

Measurement: 2a19

Data format: 1xShort

GPS Location

Service: 1860

Measurement: 2c30

Data format: 3xDouble (in the order of Latitude, Longiture, Altitude)
